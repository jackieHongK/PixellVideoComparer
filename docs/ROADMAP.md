# Product Roadmap

## North Star

Make Pixell Video Comparer the default lightweight browser workspace for visual video QA.

## Guiding Principles

- Optimize for repeated QA workflows, not generic playback
- Reduce setup time before expanding power-user controls
- Ship proof of value before broadening scope
- Keep the hosted product path as priority one
- Protect focus with milestone-based gates

## Milestone 0: Foundation Reset

Status:

- in progress

Goal:

- establish clean product structure and team operating rules

Definition of done:

- dev and prod build split is in place
- collaboration guide exists
- product strategy docs exist
- local single-file track is separated

## Milestone 1: Sharp Positioning

Goal:

- make the product understandable in under 10 seconds

Deliverables:

- revise landing/about messaging around visual QA
- define one primary ICP and one secondary ICP
- publish 3 concrete use cases
- create basic feedback CTA

Success metrics:

- users can describe the product category clearly
- inbound feedback references the intended use case

## Milestone 2: Activation Improvement

Goal:

- make first-use success fast and obvious

Deliverables:

- improved onboarding cues in dev build
- polished empty states
- clearer controls and labels
- sample/demo workflow assets
- reduced confusion around supported sources

Success metrics:

- users complete first comparison without instruction
- users export at least one frame capture in early sessions

## Milestone 3: QA Workflow Credibility

Goal:

- become meaningfully better for real QA work

Candidate features:

- annotations or comments
- saved compare sessions
- better HLS diagnostics
- comparison presets
- review evidence packaging

Success metrics:

- repeat usage from at least one target segment
- users ask for deeper workflow features instead of basic explanation

## Milestone 4: Early PMF Search

Goal:

- identify the segment with the strongest retention and clearest willingness to adopt

Deliverables:

- 10 to 15 interviews completed
- structured feedback log
- usage-based feature prioritization
- refined ICP and positioning

Success metrics:

- one segment shows repeated use
- one message angle clearly wins

## Milestone 5: Monetization Readiness

Goal:

- validate whether team workflows justify paid features

Candidate offers:

- hosted team workspace
- share links and comments
- private deployment support
- saved history and QA records

Success metrics:

- at least a few users ask for team features
- at least a few conversations suggest willingness to pay

## Next 6 Weeks

### Week 1

- lock positioning statement
- refine about page and README
- document target ICPs and JTBD

### Week 2

- create user interview script
- recruit first 5 target users
- define landing page message variants

### Week 3

- improve dev build onboarding and first-use clarity
- prepare demo assets and evidence screenshots

### Week 4

- run first feedback sessions
- log friction points and feature requests
- rank issues by activation impact

### Week 5

- ship activation improvements
- retest with new users
- compare message performance

### Week 6

- choose primary segment
- cut low-signal ideas
- set milestone 3 feature scope

## What We Should Not Do Yet

- build broad team collaboration suites too early
- chase many customer types at once
- over-invest in visual polish before activation is clear
- overbuild local-only workflows before hosted demand is validated
- add advanced features without proof that core comparison is sticky

## Decision Gates

### Gate A

After first interviews:

- is the pain frequent and expensive enough

### Gate B

After first activation fixes:

- do new users succeed without handholding

### Gate C

After early retention checks:

- which segment comes back

### Gate D

After PMF search:

- are we a utility, a workflow tool, or an emerging team product
